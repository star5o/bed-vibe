# 功能分析

## 产品定位

让用户在手机上拥有和坐在电脑前一样的 Claude Code 操作体验。

## 系统组成

- **Client**：运行在用户本地机器上的后台服务（daemon），自动感知并桥接所有 Claude Code 会话到 Server
- **Server**：提供 Web 访问 / PWA，手机通过它与 Client 通信

---

## 第一期：Claude Code

### Client 端

#### 启动与运行

- daemon 模式运行（`bv start` 启动，`bv stop` 停止）
- 利用 Claude Code 原生 hooks 机制自动感知会话，用户照常 `claude` 启动即可
- 支持开机自启（可选）
- `bv status` 查看 daemon 状态和连接情况
- `bv config` 配置 Server 地址和机器 token

> **现状：** 当前是 wrapper 模式（`bv [claude-args]` 启动），需要重构为 daemon 模式。

#### 会话感知

- 自动检测所有本地 Claude Code 会话的启动和退出
- 实时转发会话输出到 Server
- 接收 Server 下发的用户消息，注入到对应会话
- 接收 Server 下发的中断指令（等同 Esc）
- 接收 Server 下发的 slash commands，注入到对应会话
- 转发会话的 token/费用统计信息

> **现状：** 会话感知通过 hook-server + session-scanner 实现（已有），但只支持单会话。需要扩展为多会话并发管理。消息注入通过 remote-session 的 stream-json 模式实现（已有），但需要支持不中断本地终端的注入方式。

#### 远程会话

- 接收 Server 指令，在指定目录启动 headless Claude Code 进程
- 管理多个远程会话的生命周期（启动、运行、退出）
- 支持指定模型、初始 prompt
- 路径存在性检查（Server 请求验证后再启动）

> **现状：** remote-session.ts 已实现单个 headless Claude 进程管理。需要扩展为多会话并发。

#### 会话控制转发

- 接收 Server 下发的模型切换指令
- 接收 Server 下发的权限模式切换指令
- 接收 Server 下发的 effort 调整指令
- 转发文件上传到会话

> **现状：** 未实现。

### Server 端（Web / PWA）

#### 基础体验

- 白天 / 黑夜主题切换（跟随系统 / 手动）
- Markdown 渲染、代码块语法高亮
- 连接状态指示（手机↔Server↔Client 的连通性）
- PWA 支持（添加到桌面，离线 shell）

> **现状：** 主题切换已实现（ThemeProvider + Settings 页面）。Markdown 渲染已有（react-markdown + rehype-highlight）。连接状态未在 UI 展示（useSse hook 有 connected 状态但未显示）。PWA 已配置（vite-plugin-pwa）。

#### 会话列表

- 展示所有活跃会话（Claude Code 进程正在运行中的）
- 区分来源：终端开的 / 手机远程开的
- 显示会话的工作目录、模型、启动时间
- 显示会话实时状态（thinking / 执行工具 / idle / 等待输入）
- 未读标记：会话有新输出但用户未查看
- 会话结束后归入历史列表

> **现状：** Sessions 页面已实现基础列表（显示 cwd、model、status、时间）。缺少：来源区分、未读标记、活跃/历史分区。

#### 历史会话

- 搜索：按目录、时间、关键词
- 查看历史对话记录
- 会话归档 / 删除

> **现状：** 未实现。当前所有会话平铺展示，无搜索、无归档。

#### 会话操作

- 复制会话 UUID
- 复制完整接管命令（`cd /xxx && claude --resume xxx`）
- 重命名会话

> **现状：** 重命名已实现（Chat 页面内）。复制 UUID 和接管命令未实现。

#### 对话交互

- 查看实时输出（流式）
- 发送消息（与本地终端输入等价，先到先得）
- 发送文件（上传截图、参考文档等附件到会话）
- 中断 Agent 输出（等同 Esc）
- 查看历史消息（断线重连后补全）
- 输出中的图片渲染
- 消息分页加载（长对话向上滚动加载更多）

> **现状：** 实时输出、发送消息、中断（abort）、断线重连补全均已实现。缺少：文件上传、图片渲染、分页加载（API 支持 afterSeq 分页但 UI 未使用）。

#### 会话控制

- 动态切换模型（如 opus / sonnet）
- 动态切换权限模式（如 auto-accept / prompt）
- 动态调整 thinking effort
- Slash commands 触发（/compact、/clear、/model 等）
- Token / 费用显示（每个会话的消耗统计）

> **现状：** 均未实现。需要 Server 新增 API + CLI 新增指令转发。

#### 权限审批

- 展示工具调用的权限请求（工具名、参数详情）
- 审批 / 拒绝
- 审批时可选"本次会话始终允许该工具"
- 权限请求的推送通知（Agent 在等你操作时提醒）

> **现状：** 基础审批流程已实现（PermissionCard 组件 + approve/deny API + CLI 端 promise 解析）。缺少："始终允许"选项（API 有 mode/allowTools 字段但 UI 未暴露）、推送通知。

#### 新建会话

- 输入任意工作目录路径，在 Client 机器上启动新的 Claude Code 会话
- 多机器时选择目标机器
- 路径存在性检查（输入后验证目录是否存在，避免无效启动）
- 可选指定模型、初始 prompt

> **现状：** 未实现。需要 Server 新增 spawn API + CLI daemon 接收 spawn 指令。

#### 通知

- Agent 等待权限审批时推送
- Agent 任务完成时推送（从 thinking/working 变为 idle）
- 会话异常退出时推送
- 基于 Web Push，浏览器原生推送，不依赖 App 常驻

> **现状：** 未实现。需要 VAPID 密钥管理 + Service Worker push handler + Server 端推送逻辑。

#### 用户管理

- 登录 / 注册
- 多机器绑定（一个用户可以有多台 Client 机器）
- 机器在线状态显示
- 默认管理员账户
- 管理员面板：查看所有用户的使用频率、流量消耗、会话数等统计

> **现状：** 登录/注册/JWT 已实现。多机器管理已实现（Machines 页面）。缺少：机器在线状态实时显示、管理员面板/统计。

#### Client 诊断

- 查看 Client daemon 运行状态（在线/离线、版本、运行时长）
- 查看活跃进程列表
- 查看 daemon 日志（排查连接问题）

> **现状：** 未实现。需要 Server 新增诊断 API + CLI daemon 上报运行信息。
